Nova 不要求团队被单一模型或云厂商锁住。你可以在同一个编辑器里切换托管模型、本地模型和企业网关， 同时保留 VS Code 的扩展生态、语言服务、终端和源码管理。

Agent workflows

让 AI 改代码，也让你保持控制

Nova 的 Agent 先进行受限只读检查，再生成结构化计划。每一步都有计划文件、差异预览和运行记录， 适合小到中等规模的多文件修改。

1 读取上下文

只允许搜索和读取工作区内的安全路径。

2 生成计划

写入 `.nova/plans/`，先审阅再进入修改。

3 预览差异

每个文件都需要确认，降低误写风险。

4 记录结果

运行报告保存到 `.nova/runs/` 方便追踪。

模型连接测试

从侧边栏直接验证 base URL、模型 ID、密钥和自定义参数是否可用。

Run locally

从源码启动 Nova

当前项目以 fork overlay 方式开发，先准备依赖，再把 Nova 扩展和产品配置应用到 VS Code checkout。

pnpm install
pnpm setup:node
pnpm build:extension
pnpm bootstrap:vscode
pnpm apply:overlay
pnpm start:vscode -- --disable-gpu
